# Blog Section Design

**Date:** 2026-03-15
**Status:** Approved

---

## Overview

Add a blog section to the existing `things-i-learned` MkDocs Material site. The site will have three top-level sections: About (home page), TIL, and Blog. Sections share a tag system but are otherwise independent. Dark mode is enabled. Social links (GitHub, LinkedIn) appear in the footer.

---

## Goals

- Add a personal About Me home page as the site landing
- Add a Blog section using MkDocs Material's built-in blog plugin
- Reorganise TIL into a `til/` subfolder to enable clean top-level tab navigation
- Share a single tags index across both TIL and Blog
- Enable dark mode and footer social links

---

## Folder Structure

```
docs/
├── index.md                        # About Me (replaces current TIL-focused home)
├── tags.md                         # shared tag index — unchanged
│
├── til/                            # TIL section (categories moved here)
│   ├── index.md                    # TIL landing page (new)
│   ├── networking/
│   │   └── dns-ttl-...md           # existing entry, moved from docs/networking/
│   ├── security/.gitkeep
│   ├── ai-ml/.gitkeep
│   └── hardware/.gitkeep
│
└── blog/
    └── posts/                      # blog posts go here
```

**Key points:**
- `docs/index.md` is rewritten as a static About Me page
- TIL category folders move from `docs/<category>/` → `docs/til/<category>/`
- **`docs/blog/index.md` is auto-generated by the blog plugin — do NOT create it manually or the build will conflict**
- `docs/tags.md` stays at the root; the tags plugin scans all docs including both `til/` and `blog/`
- `docs/superpowers/` remains excluded from the build via `exclude_docs`
- Tab order is controlled by an explicit `nav:` block (see mkdocs.yml section)

## Updated `mkdocs.yml`

```yaml
site_name: Things I Learned
site_description: A personal knowledge log
site_url: https://markgravestock.github.io/things-i-learned/

theme:
  name: material
  palette:
    scheme: slate
  features:
    - navigation.tabs
    - navigation.indexes
    - search.suggest
    - search.highlight

plugins:
  - search
  - tags
  - meta
  - blog:
      blog_dir: blog

nav:
  - Home: index.md
  - TIL:
      - til/index.md
      - Networking: til/networking/dns-ttl-controls-how-long-resolvers-cache-records.md
  - Blog: blog/index.md
  - Tags: tags.md

extra:
  social:
    - icon: fontawesome/brands/github
      link: https://github.com/MarkGravestock
    - icon: fontawesome/brands/linkedin
      link: https://linkedin.com/in/<your-linkedin-slug>  # replace with your LinkedIn slug

exclude_docs: |
  superpowers/
```

**Changes from previous config:**
- `palette: scheme: slate` — dark mode
- `navigation.tabs` — top-level sections become header tabs
- `blog` plugin with `blog_dir: blog`
- `nav:` block — explicit tab order (Home, TIL, Blog, Tags). Without this, MkDocs sorts tabs alphabetically giving Blog before TIL
- `extra.social` — GitHub and LinkedIn footer icons

> LinkedIn: replace `<your-linkedin-slug>` with the path segment from your profile URL (e.g. `markgravestock`).

**Note on `nav:` and TIL categories:** MkDocs nav entries must point to actual markdown files — you cannot reference a directory that contains only a `.gitkeep`. Only add a category to nav when it contains at least one real `.md` entry. The pattern for adding a new category with content:

```yaml
- TIL:
    - til/index.md
    - Networking: til/networking/dns-ttl-...md  # list individual files
    - New Category: til/new-category/my-first-entry.md  # add when first entry exists
```

**Note on `blog/index.md` in nav:** The blog plugin generates this file at build time. Referencing it in `nav:` is safe — MkDocs resolves it during build.

## Page Formats

### `docs/index.md` — About Me

```markdown
# Mark Gravestock

A short bio — who you are, what you do, what you're interested in.

## Links

- [GitHub](https://github.com/MarkGravestock)
- [LinkedIn](https://linkedin.com/in/<your-slug>)

## This Site

- [TIL](til/index.md) — short notes on things I've learned
- [Blog](blog/index.md) — longer posts
```

### `docs/til/index.md` — TIL landing page

```markdown
# TIL

Short notes on things I've learned across software, hardware,
networking, security, AI/ML, and more.

Browse by category using the navigation, or search for a topic.
Use the [Tags](../tags.md) index to browse by tag.
```

### `docs/blog/posts/YYYY-MM-DD-slug.md` — Blog post

```markdown
---
date: 2026-03-15
tags:
  - networking
categories:
  - general
---

# Post Title

<!-- more -->

Full post content...

## References

- [Source](https://example.com)
```

**Blog post rules:**
- Filename: `YYYY-MM-DD-slug.md` (date prefix required by blog plugin)
- `<!-- more -->` controls excerpt cut-off on the blog listing page
- `tags` are shared with TIL entries and appear in the common `/tags/` index
- `categories` are blog-specific groupings independent of TIL categories

## Navigation Behaviour

With `navigation.tabs`, `navigation.indexes`, and the explicit `nav:` block, the tab bar renders as:

| Tab label | Source | nav: entry |
|-----------|--------|------------|
| Home | `docs/index.md` | `- Home: index.md` |
| TIL | `docs/til/index.md` + children | `- TIL: ...` section |
| Blog | `docs/blog/index.md` (auto-generated) | `- Blog: blog/index.md` |
| Tags | `docs/tags.md` | `- Tags: tags.md` |

Tab labels come from the `nav:` keys, not from page headings — so "Home" appears in the tab even though `index.md` starts with `# Mark Gravestock`.

**Adding a new TIL category:** Create a new folder under `docs/til/`, add your first entry, then add a nav entry pointing to that file. Empty folders (`.gitkeep` only) cannot be nav targets — only add the nav entry once real content exists.

```yaml
- TIL:
    - til/index.md
    - Networking: til/networking/dns-ttl-...md
    - New Category: til/new-category/first-entry.md  # ← add when content exists
```

## Migration Steps (existing content)

1. Create `docs/til/` and `docs/blog/posts/` directories
2. Move `docs/networking/` → `docs/til/networking/`
3. Move `docs/security/` → `docs/til/security/`
4. Move `docs/ai-ml/` → `docs/til/ai-ml/`
5. Move `docs/hardware/` → `docs/til/hardware/`
6. Create `docs/til/index.md` (TIL landing page)
7. Rewrite `docs/index.md` as About Me
8. Update `mkdocs.yml` with nav block, blog plugin, dark mode, and social links
9. Add a placeholder blog post or `.gitkeep` under `docs/blog/posts/` to keep the directory in git

**Do NOT create `docs/blog/index.md`** — the blog plugin auto-generates this at build time. Creating it manually will conflict with the plugin.

- Color palette customisation beyond dark mode
- Custom fonts
- RSS feed configuration (blog plugin enables this by default; can be configured later)
- Comment system
- Author profiles (blog plugin supports these; can be added later)
- Custom blog archive settings
